# 端到端自动驾驶概述

## 🎯 核心思想

**端到端自动驾驶**：从原始传感器输入直接输出控制信号，跳过传统模块化流程。

```
传统方法：感知 → 预测 → 规划 → 控制（分模块）
端到端：传感器数据 → 神经网络 → 控制指令（一体化）
```

## 📊 发展历程

### 第一阶段：直接端到端（2016-2019）
- **NVIDIA PilotNet** (2016)：CNN 直接从图像预测方向盘角度
- 问题：可解释性差，泛化能力弱

### 第二阶段：中间表示（2019-2022）
- 引入 BEV、Occupancy 等中间表示
- 提升可解释性和性能

### 第三阶段：规划导向（2022-至今）
- **UniAD** (2023)：以规划为核心统一各任务
- **VAD** (2023)：向量场景表示
- **基于基础模型**：EMMA (Waymo 2024)

## 🔄 端到端 vs 模块化

| 特性 | 模块化 | 端到端 |
|------|--------|--------|
| 架构 | 分模块独立优化 | 统一优化 |
| 误差累积 | 严重 | 减轻 |
| 可解释性 | 强 | 弱（正在改进）|
| 数据效率 | 低 | 高 |
| 部署难度 | 低 | 高 |

## 🧠 核心技术

### 1. 模仿学习 (Imitation Learning)
- 从专家演示中学习
- 行为克隆 (Behavior Cloning)
- DAgger (Dataset Aggregation)

### 2. 强化学习 (Reinforcement Learning)
- 在仿真环境中训练
- 奖励函数设计
- 模仿 + 强化结合

### 3. 传感器融合
- 相机 + LiDAR
- Transformer 融合 (TransFuser)
- BEV 融合

### 4. 时序建模
- 历史信息聚合
- 未来预测
- RNN / Transformer

## 📐 主流架构

### UniAD (2023)
```
图像 → Backbone → BEV特征
                ↓
        Track/Map/Motion/Occupancy (并行任务)
                ↓
              Planner
```
**核心**：以规划为导向，统一各感知预测任务

### TransFuser (2021-2023)
```
图像 → CNN → 图像特征 ↘
                      Transformer融合 → Planner
LiDAR → PointNet → 点云特征 ↗
```
**核心**：Transformer 跨模态融合

### ST-P3 (2022)
```
图像 → 时空特征学习 → 可视化 + 规划
```
**核心**：时空联合建模

## 💡 自动对焦应用思考

端到端思想可迁移：

1. **输入**：图像序列（而非单帧）
2. **中间表示**：场景理解、深度估计
3. **输出**：对焦位置/焦距

**优势**：
- 考虑时序信息，避免单帧抖动
- 场景理解辅助对焦策略
- 端到端优化，减少模块误差累积
