## 一、接线前的安全准备

### 1.1 工具与耗材清单

| 工具/耗材 | 用途 | 备注 |
|-----------|------|------|
| 万用表 | 测量电压、通断 | 必备 |
| 防静电手环 | 保护MCU/驱动芯片 | 强烈建议 |
| 热缩管 + 热风枪 | 绝缘裸露焊点 | 必备 |
| 杜邦线（公母/母母） | 信号线连接 | 建议用不同颜色区分 |
| 硅胶线 14~18AWG | 电机动力线 | 必须用硅胶线，耐高温 |
| 电流表（串联） | 监测实际电流 | 调试阶段强烈建议 |
| 限流电阻/灯泡（假负载） | 首次上电保护 | 见"首次上电"章节 |

### 1.2 颜色规范（强烈建议遵守）

```
红色  → 正极电源（+24V / +5V / +3.3V）
黑色  → 负极/GND
黄色  → 电机相线 A
绿色  → 电机相线 B
蓝色  → 电机相线 C
白色  → 编码器信号线
橙色  → CAN_H
紫色  → CAN_L
```

## 三、各模块详细接线

### 3.1 电源接线（24V 主电源）

```
开关电源 +V  ──→  驱动板 VM（电机电源正极）
开关电源 GND ──→  驱动板 GND
开关电源 GND ──→  MCU GND（共地！必须）
```

> ⚠️ **共地原则**：所有模块的 GND 必须连接在一起，否则信号参考电位不同，通信会出错甚至损坏器件。

### 3.2 MCU → 驱动板接线（以 STM32F4 + SimpleFOC Shield 为例）

SimpleFOC Shield v2 直接插在 Arduino/STM32 Nucleo 上，无需额外接线。

若使用独立驱动板（如 DRV8302/DRV8305），参考以下接线：

| 驱动板引脚 | MCU 引脚 | 说明 |
|-----------|---------|------|
| INH_A | PA8 (TIM1_CH1) | A相高侧 PWM |
| INH_B | PA9 (TIM1_CH2) | B相高侧 PWM |
| INH_C | PA10 (TIM1_CH3) | C相高侧 PWM |
| INL_A | PB13 | A相低侧 PWM（6路PWM模式）|
| INL_B | PB14 | B相低侧 PWM |
| INL_C | PB15 | C相低侧 PWM |
| EN/FAULT | PB0 | 使能/故障引脚 |
| SO1 | PA0 (ADC) | 电流采样1 |
| SO2 | PA1 (ADC) | 电流采样2 |

### 3.3 AS5600 磁编码器接线

| AS5600 引脚 | 连接目标 | 说明 |
|------------|---------|------|
| VCC | 3.3V 或 5V | 供电（查芯片手册确认） |
| GND | GND | 共地 |
| SDA | MCU SDA（如 PB7） | I2C 数据线 |
| SCL | MCU SCL（如 PB6） | I2C 时钟线 |
| DIR | GND 或 VCC | 旋转方向选择 |
| OUT | 不接（I2C模式） | 模拟/PWM输出，I2C模式不用 |

> 💡 **I2C 上拉电阻**：SDA 和 SCL 需要接 4.7kΩ 上拉电阻到 3.3V。AS5600 模块板通常已内置，裸芯片需自行添加。

> 💡 **磁铁安装**：AS5600 需要在芯片正上方 0.5~3mm 处安装直径 6mm 的轴向充磁圆形磁铁，磁铁必须与电机轴同轴旋转。

### 3.4 电机三相线接线

```
驱动板 PHASE_A ──→ 电机 A相（任意一相，后续通过校准确定方向）
驱动板 PHASE_B ──→ 电机 B相
驱动板 PHASE_C ──→ 电机 C相
```

> ⚠️ 三相线接错不会损坏电机，但电机可能反转或无法正常运行。SimpleFOC 的 `motor.initFOC()` 会自动校准相序，无需手动对应。

## 四、安全保护配置

### 4.1 硬件过流保护

**方案A：驱动芯片内置保护（推荐）**

DRV8302/DRV8305 等芯片内置过流保护（OCP），通过电阻设置阈值：

```
// DRV8302 过流阈值设置（通过 GAIN 和 OC_ADJ 引脚）
// OC_ADJ 引脚接不同电阻对应不同阈值，参考芯片手册 Table 1
// 推荐初始设置：限流 10A（对应 OC_ADJ = 0.5V）
```

**方案B：串联保险丝（兜底保护）**

在电源正极串联合适规格的保险丝：
- 5010 航模电机（空载）：串联 **5A 慢熔保险丝**
- GM6020 云台电机：串联 **10A 慢熔保险丝**

### 4.2 SimpleFOC 软件限制参数

在代码初始化阶段务必设置以下保护参数：

```cpp
// ===== 安全参数配置（必须在 motor.init() 之前设置）=====

// 1. 电压限制（最重要！防止过压损坏电机）
motor.voltage_limit = 6.0;       // 单位：V，建议从低值开始（电源电压的25%）
motor.voltage_sensor_align = 3.0; // 校准时使用的电压，更低更安全

// 2. 速度限制（防止飞车）
motor.velocity_limit = 20.0;     // 单位：rad/s，约 190 RPM，初始调试用

// 3. 电流限制（如果使用电流传感器）
motor.current_limit = 2.0;       // 单位：A，初始调试建议 ≤ 2A

// 4. PID 输出限制（防止积分饱和）
motor.PID_velocity.output_ramp = 300; // 速度环输出斜率限制
motor.P_angle.output_ramp = 100;      // 位置环输出斜率限制
```

### 4.3 软件急停实现

```cpp
// 在主循环中添加急停检测
#define ESTOP_PIN 2  // 急停按钮接 D2，另一端接 GND

void setup() {
    pinMode(ESTOP_PIN, INPUT_PULLUP);
    // ... 其他初始化
}

void loop() {
    // 急停检测（低电平触发）
    if (digitalRead(ESTOP_PIN) == LOW) {
        motor.disable();  // 立即关闭电机
        Serial.println("急停触发！电机已关闭");
        while(1);  // 停止所有操作，需手动复位
    }

    motor.loopFOC();
    motor.move(target);
}
```

## 五、首次上电流程（防炸机步骤）

### 第一步：断电检查

- [ ] 所有 GND 已共地
- [ ] 电源正负极未短路（用万用表测量驱动板 VM 和 GND 之间阻值，应 > 1kΩ）
- [ ] 电机三相线无短路（相间阻值应相等，约几欧姆）
- [ ] 编码器供电电压正确（3.3V 或 5V，不可接错）
- [ ] MCU 与驱动板共地已连接

### 第二步：限流上电（灯泡法）

在电源正极串联一个 **60W 白炽灯泡**（或 10Ω/10W 电阻）作为限流器：

```
开关电源 +24V → [60W灯泡] → 驱动板 VM
```

- 正常情况：灯泡微亮或不亮（空载电流很小）
- 短路情况：灯泡全亮，立即断电检查

### 第三步：编码器验证

上电后先只运行编码器读取代码，验证数值正确：

```cpp
#include <SimpleFOC.h>

MagneticSensorI2C sensor = MagneticSensorI2C(AS5600_I2C);

void setup() {
    Serial.begin(115200);
    Wire.begin();
    sensor.init();
    Serial.println("编码器初始化完成");
}

void loop() {
    sensor.update();
    Serial.print("角度: ");
    Serial.print(sensor.getAngle());
    Serial.print(" rad\t速度: ");
    Serial.println(sensor.getVelocity());
    delay(100);
}
```

**验证标准**：
- 手动旋转电机轴，角度值应连续变化（无跳变）
- 顺时针旋转角度增大（或减小，方向一致即可）
- 速度值在静止时接近 0

### 第四步：开环低速测试

```cpp
// 开环速度控制测试（不需要编码器反馈）
motor.controller = MotionControlType::velocity_openloop;
motor.voltage_limit = 3.0;  // 极低电压，安全第一

// 目标速度：1 rad/s（约 10 RPM，肉眼可见缓慢旋转）
motor.move(1.0);
```

**观察要点**：
- 电机应缓慢平稳旋转
- 无异响（嗡嗡声正常，金属摩擦声异常）
- 驱动板无过热（手摸不烫）
- 电流表读数 < 1A（空载）

### 第五步：闭环校准

```cpp
// 运行 motor.initFOC() 进行自动校准
// 校准过程：电机会缓慢旋转一圈，确定编码器零点和相序
// 正常现象：电机缓慢转动约 2-3 秒后停止
motor.initFOC();
```

## 六、常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 电机不转，无声音 | 使能引脚未拉高 / 代码未调用 `motor.enable()` | 检查 EN 引脚，添加 `motor.enable()` |
| 电机抖动不旋转 | 编码器方向与电机方向不匹配 | 在 `BLDCMotor` 构造函数中修改极对数符号 |
| 电机转动但速度不稳 | PID 参数不合适 | 降低 Kp，增大 Ki |
| 驱动板发热严重 | 电流过大 / 散热不足 | 降低 `voltage_limit`，加装散热片 |
| 编码器读数跳变 | I2C 信号干扰 / 磁铁偏心 | 加磁珠滤波，重新对准磁铁 |
| 上电后保险丝熔断 | 电源正负极接反 / 短路 | 断电检查接线 |
| `initFOC()` 失败 | 编码器未正确初始化 | 先单独验证编码器读数 |
